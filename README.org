#+title: Completionist: Persistent Completion Buffers for Dynamic UI
#+author: Overdr0ne
#+language: en

Completionist is a framework for building *persistent, filterable list interfaces* in Emacs.
It is a fork of [[https://github.com/minad/vertico][Vertico]] that has diverged entirely in purpose.

#+toc: headlines 3

* What Completionist Is Not

Completionist does *not* extend ~completing-read~ or the minibuffer. It has no integration
with ~find-file~, ~switch-to-buffer~, ~execute-extended-command~, or any standard Emacs
completion command. For that, use Vertico.

* What Completionist Is

Completionist creates *persistent buffers* that act as filterable UI widgets. These buffers
stay open, can be positioned anywhere in your frame, support multiple concurrent instances,
and update automatically via hooks or timers.

Think of it as a toolkit for building reactive side panels, tab bars, process monitors, or
any dynamic list interface — not as a minibuffer completion replacement.

| Feature | Vertico | Completionist |
|--------------------------+---------+---------------|
| Target | ~completing-read~ | Persistent buffer widgets |
| Lifetime | Ephemeral — closes after selection | Persistent — stays open |
| Instances | One at a time | Multiple concurrent |
| Updates | On user input only | Via hooks, timers, or manually |
| API | Hooks into ~completing-read~ | ~completionist--complete~ |

* Core API

#+begin_src emacs-lisp
(completionist--complete PROMPT COLLECTOR HANDLER BUFFER-NAME ACTION
                         &optional UNFOCUSP DISPLAY-MODE UPDATE-HOOKS UPDATE-INTERVAL
                         HISTORY SORT-FN)
#+end_src

- *PROMPT*: String shown before the input area
- *COLLECTOR*: Function returning a fresh list of candidate strings; called on each keystroke
- *HANDLER*: Function called with the selected candidate when the user presses =RET=
- *BUFFER-NAME*: Persistent buffer name — reused if it already exists
- *ACTION*: ~display-buffer~ action controlling window placement and size
- *UNFOCUSP*: If non-nil, don't steal focus from the current buffer
- *DISPLAY-MODE*: ~nil~ (vertical, default), ~'flat~ (horizontal), ~'grid~, or ~(cons arrange-fn display-fn)~
- *UPDATE-HOOKS*: List of hooks that trigger automatic refresh
- *UPDATE-INTERVAL*: Seconds between timer-based refreshes, or ~nil~
- *HISTORY*: Initial history list for this widget
- *SORT-FN*: Sorting function for this widget

Force a manual refresh of any completionist buffer:

#+begin_src emacs-lisp
(when-let ((buf (get-buffer "*my-widget*")))
  (completionist--exhibit buf))
#+end_src

* Examples

** Perspective tab bar

#+begin_src emacs-lisp
(defun my-persp-tabs ()
  (interactive)
  (completionist--complete
    "persp:"
    #'persp-names
    #'persp-switch
    " *persps*"
    '((display-buffer-in-side-window)
      (side . top)
      (window-height . 1))
    t      ; unfocusp - don't steal focus
    'flat  ; horizontal layout
    '(persp-created-hook persp-killed-hook persp-renamed-hook)))
#+end_src

** Live process monitor

#+begin_src emacs-lisp
(defun my-process-monitor ()
  (interactive)
  (completionist--complete
    "processes:"
    (lambda ()
      (mapcar (lambda (p)
                (format "%-20s %s" (process-name p) (process-status p)))
              (process-list)))
    (lambda (line)
      (when (string-match "^\\(\\S-+\\)" line)
        (delete-process (get-process (match-string 1 line)))))
    "*processes*"
    '((display-buffer-in-side-window)
      (side . right)
      (window-width . 40))
    t nil                       ; unfocusp, default display-mode
    '(comint-exec-hook) 5.0))   ; hook updates + 5s timer for exits
#+end_src

** Multiple concurrent widgets

#+begin_src emacs-lisp
;; Buffer list on the left (vertical)
(completionist--complete
  "buffers:"
  (lambda () (mapcar #'buffer-name (buffer-list)))
  #'switch-to-buffer
  "*comp-buffers*"
  '((display-buffer-in-side-window) (side . left) (window-width . 30))
  t) ; unfocusp

;; Perspective tabs at the top (flat)
(completionist--complete
  "persp:"
  #'persp-names
  #'persp-switch
  " *persps*"
  '((display-buffer-in-side-window) (side . top) (window-height . 1))
  t 'flat
  '(persp-created-hook persp-killed-hook persp-renamed-hook))
#+end_src

* Automatic Updates
:properties:
:custom_id: automatic-updates
:end:

Completionist buffers can update automatically without user interaction.

| Strategy | When to use | Example use cases |
|----------+-------------+-------------------|
| Hook-based | Data has change events | Perspectives, buffer list, themes |
| Timer-based | No hooks available | Clocks, system stats, network |
| Hybrid (both) | Critical data or unreliable hooks | Processes, file watchers |

#+begin_src emacs-lisp
;; Hook-based: update when perspectives change
(completionist--complete "persp:" #'persp-names #'persp-switch " *persps*"
                         '((display-buffer-in-side-window) (side . top) (window-height . 1))
                         t 'flat
                         '(persp-created-hook persp-killed-hook) ; hooks
                         nil)                                     ; no timer

;; Timer-based: update every second (clock)
(completionist--complete "time:" (lambda () (list (format-time-string "%H:%M:%S"))) #'ignore
                         "*clock*"
                         '((display-buffer-in-side-window) (side . top) (window-height . 1))
                         t nil nil 1.0) ; 1 second interval

;; Hybrid: hooks for starts, timer for exits
(completionist--complete "processes:" #'process-list-names #'delete-process
                         "*processes*"
                         '((display-buffer-in-side-window) (side . right) (window-width . 40))
                         t nil '(comint-exec-hook) 5.0)
#+end_src

* Display Modes

Specify via the ~display-mode~ parameter:

- ~nil~ — vertical list (default)
- ~'flat~ — horizontal, one-line tab bar style
- ~'grid~ — multi-column grid
- ~(cons arrange-fn display-fn)~ — fully custom layout

Extension keymaps (~completionist-flat-map~, ~completionist-grid-map~) are merged into the
buffer's local keymap automatically, remapping arrow keys to layout-aware navigation.

* Key Bindings

All completionist buffers share ~completionist-map~, derived from ~minibuffer-local-map~:

- ~beginning-of-buffer~, ~minibuffer-beginning-of-buffer~ → ~completionist-first~
- ~end-of-buffer~ → ~completionist-last~
- ~scroll-down-command~ → ~completionist-scroll-down~
- ~scroll-up-command~ → ~completionist-scroll-up~
- ~next-line~, ~next-line-or-history-element~ → ~completionist-next~
- ~previous-line~, ~previous-line-or-history-element~ → ~completionist-previous~
- ~forward-paragraph~ → ~completionist-next-group~
- ~backward-paragraph~ → ~completionist-previous-group~
- ~exit-minibuffer~ → ~completionist-execute~ (calls handler, keeps buffer open)
- ~kill-ring-save~ → ~completionist-save~
- =M-RET= → ~completionist-exit-input~
- =TAB=, =C-l= → ~completionist-insert~
- =C-q= → ~quit-window~

In flat mode, =←=/=→= navigate candidates. In grid mode, =←=/=→= move between columns.

* Configuration

#+begin_src emacs-lisp
;; Number of visible candidates (calculated per-buffer based on window size)
(setq completionist-count 10)

;; Enable cycling at list boundaries
(setq completionist-cycle t)

;; Scroll margin
(setq completionist-scroll-margin 2)

;; Default sort function
(setq completionist-sort-function #'completionist-sort-history-length-alpha)
#+end_src

Available sort functions: ~completionist-sort-history-length-alpha~ (default),
~completionist-sort-history-alpha~, ~completionist-sort-length-alpha~,
~completionist-sort-alpha~, or ~nil~ to preserve collector order.

Each widget can have its own independent sort function and history via the ~SORT-FN~
and ~HISTORY~ parameters.

* Extensions
:properties:
:custom_id: extensions
:end:

Extensions provide alternative display layouts. They are activated *per-buffer* via
the ~display-mode~ parameter — never as global modes. Both are loaded automatically
by ~completionist.el~.

- [[file:extensions/completionist-flat.el][completionist-flat]]: Horizontal layout (~'flat~)
- [[file:extensions/completionist-grid.el][completionist-grid]]: Grid/column layout (~'grid~)

* Complementary Packages

These packages integrate naturally with completionist's candidate display and annotations:

- [[https://github.com/minad/marginalia][Marginalia]]: Rich annotations alongside candidates
- [[https://github.com/oantolin/orderless][Orderless]]: Flexible space-separated filtering of candidates
- [[https://github.com/oantolin/embark][Embark]]: Actions on candidates in any context

* Installation

Completionist is designed to be loaded as a local package:

#+begin_src emacs-lisp
(use-package completionist
  :straight (completionist :local-repo "path/to/completionist"
                           :files ("*.el"
                                   "extensions/completionist-flat.el"
                                   "extensions/completionist-grid.el")))
#+end_src
